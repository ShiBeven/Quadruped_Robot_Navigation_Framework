## 2. TF 链替换

### 2.1 当前 TF 链

```
map → odom → base_footprint → base_link → body → front_mid360
```

### 2.2 你需要给我的

仿真验证通过后，把你最终确定的 TF 链按以下格式给我：

```
base_footprint → base_link:  [x, y, z, roll, pitch, yaw]
base_link → body:            [x, y, z, roll, pitch, yaw]
body → front_mid360:         [x, y, z, roll, pitch, yaw]
body → fl_hip:               [x, y, z, roll, pitch, yaw]
body → fr_hip:               [x, y, z, roll, pitch, yaw]
body → rl_hip:               [x, y, z, roll, pitch, yaw]
body → rr_hip:               [x, y, z, roll, pitch, yaw]
```

- 如果某两个 frame 之间应该是 identity（全零），直接写 `[0, 0, 0, 0, 0, 0]`
- 如果某些 frame 不需要（比如不用足端坐标系），直接说"删掉"
- 如果需要新增 frame（比如加了深度相机），同样用上述格式追加

### 2.3 我会改什么

收到你的 TF 链后，我会修改 **1 个文件**：

| 文件 | 改动位置 | 改动内容 |
|------|----------|----------|
| `src/navigation/nav_bringup/description/quadruped.urdf` | 每个 `<joint>` 的 `<origin>` 标签 | 将 `xyz` 和 `rpy` 替换为你给的值 |

不需要改任何 C++ 代码或 YAML 配置——所有模块通过 TF 查找，不硬编码 frame_id。

## 11. 仿真环境对接

### 11.1 当前状态

`nav2_loopback_sim` 是纯 2D 回路仿真（无物理），对四足的适用性有限。你可能需要对接有物理引擎的仿真器。

### 11.2 你需要决定的

| 选项 | 说明 |
|------|------|
| Isaac Gym / Isaac Sim | NVIDIA 生态，适合 RL 训练步态 |
| MuJoCo | 开源，轻量，适合 MPC/WBC |
| Gazebo (Ignition) | ROS 原生支持最好，已有 `ign_sim_pointcloud_tool` |
| 继续用 loopback_sim | 快速验证导航逻辑（不需要物理） |

### 11.3 我会做什么

根据你的选择，我可能需要：
- 写仿真 bridge 节点（仿真 odom → ROS odometry，仿真 TF → ROS TF）
- 配置仿真传感器插件（LiDAR + IMU）
- 如果使用 Gazebo：对接 `ign_sim_pointcloud_tool`
- 如果使用 Isaac/MuJoCo：可能需要写一个薄的 ROS 2 wrapper

> **提示**: 无论用哪个仿真器，导航框架的输入接口不变（`odometry` + TF + `sensor_scan`），输出接口不变（`cmd_vel`）。只需要一个 bridge 把仿真数据灌进来。

## 12. 编译验证命令

每次改完配置文件后，**不需要完整 colcon build**（配置文件的修改不需要重新编译）。但以下情况需要重新编译：

### 12.1 需要编译的情况

| 改动类型 | 编译命令 |
|----------|----------|
| 改了 URDF | 不需要编译（URDF 是运行时加载的） |
| 改了 YAML | 不需要编译（参数是运行时加载的） |
| 改了 launch.py | 不需要编译（Python 是解释执行的） |
| 改了 BT XML | 不需要编译（XML 是运行时加载的） |
| 改了 C++ 源码 | 需要编译 |
| 改了 CMakeLists.txt | 需要编译 |

### 12.2 编译命令

```bash
# 完整编译
cd W:/Basic_Navigatio_Framework
colcon build --symlink-install --cmake-args -DCMAKE_BUILD_TYPE=Release

# 只编译单个包
colcon build --symlink-install --packages-select nav_bringup

# 编译后 source
source install/setup.bash
```

### 12.3 运行命令

```bash
# 定位模式（需要地图 + 先验 PCD）
ros2 launch nav_bringup legged_localization_launch.py \
  map:=/path/to/map.yaml \
  prior_pcd:=/path/to/prior.pcd \
  params_file:=src/navigation/nav_bringup/config/nav2_params.legged.yaml

# 导航模式（需要先启动定位）
ros2 launch nav_bringup legged_navigation_launch.py \
  params_file:=src/navigation/nav_bringup/config/nav2_params.legged.yaml

# SLAM 模式
ros2 launch nav_bringup legged_slam_launch.py \
  params_file:=src/navigation/nav_bringup/config/nav2_params.legged.yaml

# 仿真模式
ros2 launch nav_bringup legged_navigation_launch.py \
  use_sim_time:=true \
  params_file:=src/navigation/nav_bringup/config/nav2_params.legged_sim.yaml
```
